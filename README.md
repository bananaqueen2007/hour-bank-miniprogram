# 工时银行微信小程序使用手册

## 项目简介
“工时银行”是一款基于微信小程序和云开发的工时管理工具，旨在解决校园/团队中工时统计线下记录繁琐、查询不便、数据易丢失的问题。  
**核心功能**：工时录入（新增/累加）、学号精准查询、全记录浏览（测试用），实现工时数据线上化、安全存储与高效管理。

## 快速开始

### 1. 环境准备
- 下载并安装[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- 注册微信小程序，获取 **AppID**（本项目已配置：`wx4b9897923392126d`）
- 在开发者工具中导入项目文件夹，确保AppID正确

### 2. 云开发环境配置
1. 在开发者工具中点击“云开发”按钮，开通云环境。
2. 创建数据库集合 **`hours`**（或在首次使用录入功能时通过代码自动创建）。
3. 修改云函数 `quickstartFunctions` 的 `config.json`，确保权限声明包含 `wxacode.get`（如需生成小程序码）。
4. 将云函数上传并部署到云环境。

### 3. 数据库权限设置
在云开发控制台 – 数据库 – `hours` 集合中，设置权限为：
- **所有用户可读，仅创建者可写**（若允许用户新增/修改自己的记录，可根据实际调整）
- 如需更精细控制，可改用云函数操作数据库（本项目前端直接调用数据库，需保证权限开放）

## 核心功能使用方法

### 🔹 录入工时（页面：`pages/input/input`）
1. 在底部导航栏或首页点击“录入”进入录入页面。
2. 填写以下信息：
   - **姓名**（必填，将自动去除首尾空格）
   - **学号**（必填，用于唯一标识）
   - **工时数**（必填，正数，支持小数）
3. 点击“提交”按钮：
   - 若学号**不存在**，则创建新记录，工时设为输入值。
   - 若学号**已存在**，则自动**累加**工时（原有工时 + 输入值）。
4. 操作结果会通过弹窗提示成功或失败（如网络异常、参数错误）。

### 🔹 查询工时（页面：`pages/query/query`）
1. 进入“查询”页面。
2. **按学号查询**：
   - 输入学号，点击“查询”。
   - 若存在，显示该学号的姓名、当前总工时、最近更新时间。
   - 若不存在，提示“无此学号记录”。
3. **查看所有记录**（测试用）：
   - 点击“全部记录”按钮，可列出`hours`集合中所有数据（包括_id、姓名、学号、工时等）。
   - 支持长按单条记录删除（需二次确认）。

### 🔹 其他功能（云函数扩展）
本项目还包含一个多功能云函数 `quickstartFunctions`，可通过前端调用实现：
- **获取用户OpenID**：用于身份识别。
- **生成小程序码**：生成带参数的小程序码并上传至云存储，可用于分享或海报。
这些功能未集成到主页面，开发者可根据需求自行调用。

## 注意事项
- **首次使用**：需确保云环境已初始化（`app.js` 中已调用 `wx.cloud.init()`）。
- **学号唯一性**：系统以学号作为唯一标识，请勿重复注册不同姓名。
- **数据备份**：云数据库自动备份，无需担心数据丢失。
- **错误处理**：所有操作均有友好的错误提示，如网络断开、权限不足等。
- **版本管理**：项目使用Git进行版本控制，分支规范为 `feature/*` → `develop` → `main`，通过Pull Request合并。
